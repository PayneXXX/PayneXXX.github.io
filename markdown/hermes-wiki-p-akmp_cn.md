# Hermes-wiki-P/AKMP

> 一套面向人类与 AI Agent 的双轨知识库体系：人类用 PKM 保持轻量、自由、联想式记录；Agent 用 AKM 读取结构化、可检索、可维护的机器优先知识。

## 背景

普通个人知识库在引入 Agent 后容易出现两个问题：

- 人看的库被机器写坏：大量机器生成文件、技能编号和结构化页面污染 Obsidian 日常阅读体验。
- 机器吃的库不结构：Agent 每次执行任务都全量加载材料，上下文爆炸，也容易重复造轮子。

Hermes-wiki-P/AKMP 的核心方案是物理隔离：把人类知识库和 Agent 知识库拆成两套 Vault，但在需要时通过工作流互相同步。

## 双库结构

| Vault | 面向对象 | 定位 |
| --- | --- | --- |
| Penn_PKM | 人类 | 放在 iCloud，面向手机、电脑多端访问，轻量、自由、联想式 |
| Hermes_AKM | Agent 集团 | 放在本地，Git 管理，结构化、可检索、机器优先 |

PKM 保留人的思考、灵感和阅读路径；AKM 负责承载 Agent 可以稳定调用的知识、技能、模板和项目上下文。

## 核心机制

### Raw / Compiled 分层

Raw 层保存不可变原材料，例如文章、论文、对话记录和图片。Agent 只读不改，保证溯源可追。

Compiled 层保存从 Raw 加工后的技能、知识、概念页面，是 Agent 实际执行任务时调用的内容。

### 地图按需加载

Agent 不会一次性读取所有地图，而是按层级进入：

1. 全局入口：极小的 AKM_Guide。
2. 项目地图：只读取当前项目的 MOC。
3. 具体文件：按需加载 Skills、Knowledge 或 Context。

这样可以控制上下文污染，让 Agent 在任务中只看到真正相关的信息。

### 技能上浮

如果某个项目级技能复用率超过 50%，就从项目空间晋升到通用技能库，原位置保留 Stub 重定向，保证链接不断裂。

这个机制让知识系统自然沉淀：局部经验先在项目内试用，成熟后变成全局能力。

### Lint 体检

AKM 定期检查孤儿页面、断链、过期内容、标签污染、矛盾标记、Frontmatter 合法性和页面膨胀。

Lint 的目标不是追求形式洁癖，而是让知识库长期可用，避免随着规模增长变成无法维护的文件堆。

## 与 LLM Wiki 的关系

这个系统受 Karpathy 的 LLM Wiki 模式启发，继承了 Raw -> Compiled -> Schema、Orientation Ritual、Append-only Log、Lint 体检和 Page Threshold 等思想。

不同之处在于，它进一步强调双库隔离、Agent 专属知识管理、技能上浮和 ID Registry，更适合 Hermes / OpenClaw 这类 Agent 工具集。

## 项目价值

Hermes-wiki-P/AKMP 不是单纯的 Obsidian 模板，而是一套面向 AI Building 的知识基础设施。它让个人阅读、项目实践和 Agent 执行之间形成可持续循环：

- 人类在 PKM 中保持自由输入和思考。
- Agent 在 AKM 中获得规范入口和可调用知识。
- 项目中产生的 SOP、技能和上下文可以被沉淀。
- 复用率高的能力逐步上浮为通用技能。
- Lint 和 Log 保证系统长期可追踪、可维护。

它的目标是让知识不只是“被记录”，而是能进入下一次任务执行，成为人机协作系统中的真实能力。
